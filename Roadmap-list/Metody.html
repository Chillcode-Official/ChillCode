Metody Języka C#

Metody w języku C# to fragmenty kodu, które mogą być wywoływane wielokrotnie w różnych miejscach programu. Każda metoda posiada nazwę i może mieć listę argumentów (parametrów). Metoda Main jest punktem wejścia dla każdej aplikacji języka C# i jest wywoływana przez środowisko uruchomieniowe języka wspólnego (CLR) po uruchomieniu programu.

Metody mogą zwracać wartość lub też nie w przypadku metod void. 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {
            HelloWorld();  // Wywołanie metody HelloWorld
        }

        static void HelloWorld()  // Metoda wyświetlająca tekst na ekranie
        {
            Console.WriteLine("Hello World");
        }
    }
}

//output: Hello World

Moglibyśmy również stworzyć metodę, która przyjmuje jakiś argument i zwraca wartość 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {
            int suma = ObliczSume(5, 7);

            Console.WriteLine("Suma wynosi:" + suma);
        }

        static int ObliczSume(int x, int y)
        {
            return x + y;
        }
    }
}

//output: Suma wynosi: 12

Wywoływanie Metod

Aby wywołać metodę w języku C#, należy napisać jej nazwę oraz nawiasy i ewentualnie podać argumenty (jeśli metoda ich oczekuje). Jeśli metoda zwraca jakąś wartość, to możemy ją przypisać do zmiennej lub wykorzystać bezpośrednio w wyrażeniu.

Rodzaje wywoływania metod w następujący sposób: 

                
int suma = ObliczSume(6, 8); // Wywołanie metody i przypisanie rezultatu do zmiennej suma


int a = 5;
int b = 6;
int c = ObliczSume(a, b); // Wywołanie metody i przypisanie rezultatu do zmiennej c


int d = ObliczSume(4 + 1, 5 * 4); // Wywołanie metody i przekazanie wyników obliczeń jako argumenty


int e = 7;
int f = 8;
int g = ObliczSume(e++, f--);

// Wywołanie metody i przekazanie zwiększonej i zmniejszonej o 1 wartości zmiennych e i f jako argumenty

Sposoby wywoływania metod przy użyciu argumentu typu string. 

                
WypiszTekst("Witaj, świecie!");  // Wywołanie metody i przekazanie ciągu znaków jako argument


string tekst = "To jest jakiś tekst";
WypiszTekst(tekst);  // Wywołanie metody i przekazanie zmiennej jako argument


WypiszTekst("Liczba " + 42);  // Wywołanie metody i przekazanie wyniku konkatenacji jako argument

Przykład prostego kodu z użyciem metod. 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {
            int wiek = 15;

            int dostep = SprawdzanieWieku(wiek);

            Console.WriteLine(dostep);
        }

        static int SprawdzanieWieku(int wiek)
        {
            if (wiek >= 18)
            {
                Console.WriteLine("Możesz wejść na impreze (-:");
            }
            else
            {
                Console.WriteLine("Jesteś za młody na impreze dzieciaku, pobaw się w piaskownicy (;");
            }

            return wiek;
        }
    }
}

//output: Jesteś za młody na impreze dzieciaku, pobaw się w piaskownicy (; 15

Rekurencyjne Wywoływanie Metod

Rekurencyjne wywoływanie metod to sytuacja, w której metoda wywołuje sama siebie. Jest to pewien sposób programowania, który może być używany w sytuacjach, gdy problem, który chcemy rozwiązać, da się podzielić na mniejsze części, które są podobne do całości.

W poniższym przykładzie mamy metodę Silnia, która rekurencyjnie oblicza silnię z podanej liczby n. 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {
            int wynik = Silnia(5);  // Obliczenie silni z 5

            Console.WriteLine("Silnia z 5 wynosi: " + wynik);  // Wyświetlenie wyniku
        }

        // Metoda rekurencyjna obliczająca silnię
        static int Silnia(int n)
        {
            if (n == 0)
            {
                // Dla n = 0 silnia wynosi 1
                return 1;
            }
            else
            {
                // Dla n > 0 silnia wynosi n * silnia z (n - 1)
                return n * Silnia(n - 1);
            }
        }
    }
}

//output: 120   //Dlatego że 5 * 4 * 3 * 2 * 1 = 120

Dla lepszego zrozumienia spróbujmy prześledzić, jak wygląda obliczenie silni z 5 w pod punktach

Uwaga

Jeśli n jest równe 0, to metoda zwraca 1 (ponieważ silnia z 0 wynosi 1). W przeciwnym razie metoda zwraca n * silnię z (n - 1).

  1. Wywołanie metody Silnia(5)

  2. Sprawdzenie warunku n == 0, który jest nieprawdziwy

  3. Zwrócenie wyniku 5 * Silnia(4)

  4. Wywołanie metody Silnia(4)

  5. Sprawdzenie warunku n == 0, który jest nieprawdziwy

  6. Zwrócenie wyniku 4 * Silnia(3)

  7. Wywołanie metody Silnia(3)

  8. Sprawdzenie warunku n

Przekazywanie Parametrów Do Metod

W języku C# istnieją trzy sposoby przekazywania parametrów do metod: przez wartość, przez referencję oraz przez wartość i referencję. Każdy z tych sposobów ma swoje wady i zalety i może być używany w zależności od potrzeb programu.

Podczas wywoływania metody z parametrami najpierw te parametry należy przekazać do metody.

Parametry typu Wartościowego

Parametry typu wartościowego (value types) to parametry przekazywane do metody przez wartość. Oznacza to, że w momencie wywołania metody, do jej parametrów zostają skopiowane wartości argumentów.

Uwaga

Wewnątrz metody mamy do czynienia z kopią argumentu, a nie z oryginalnym obiektem.

Uwaga

Zmiany wprowadzone w obiekcie wewnątrz metody nie wpływają na oryginalny obiekt poza metodą.

  • Typy proste (primitive types) takie jak int floatdoublebool

  • Struktury (structs) - to tzw. typy wartościowe złożone, które mogą zawierać pola metodykonstruktory

  • Typy wyliczeniowe (enums) - to typy wartościowe służące do reprezentowania zbioru skończonych wartości.

Jak widać na poniższym przykładzie zmiany wartości wewnątrz metody nie mają żadnego wpływu na wartości przekazane do metody. 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {
            // Przekazanie zmiennej typu int wartość 100
            int a = 100;

            Console.WriteLine("Przed wywołaniem metody: a = {0}", a);

            Zmien(a);

            Console.WriteLine("Po wywołaniu metody: a = {0}\n", a);

            // Przekazanie zmiennej typu bool przez wartość
            bool c = true;

            Console.WriteLine("Przed wywołaniem metody: c = {0}", c);

            Zmien(c);

            Console.WriteLine("Po wywołaniu metody: c = {0}", c);
        }
        static void Zmien(int x)
        {
            x++;
        }
        static void Zmien(bool x)
        {
            x = !x;
        }
    }
}

//output:

Przed wywołaniem metody: a = 100
Po wywołaniu metody: a = 100

Przed wywołaniem metody: c = True
Po wywołaniu metody: c = True

Parametry Typu Referencyjnego

Parametry typu referencyjnego (reference types) to parametry przekazywane do metody przez referencję. Oznacza to, że w momencie wywołania metody, do jej parametrów przekazywany jest adres obiektu w pamięci.

Uwaga

Wewnątrz metody mamy bezpośredni dostęp do oryginalnego obiektu i jakiekolwiek zmiany wprowadzone w obiekcie wewnątrz metody wpływają na oryginalny obiekt poza metodą.

Uwaga

Parametr referencyjny jest definiowany przez użycie słowa kluczowego ref.

  • Typy złożone complex types - to obiekty tworzone przy użyciu klas, które mogą zawierać pola, konstruktory metody. Do typów złożonych zaliczamy między innymi: klasyinterfejsy delegaty

  • Typy dynamiczne dynamic types - to typy, których typ jest rozpoznawany w czasie wykonywania programu, a nie w czasie kompilacji. Do typów dynamicznych zaliczamy między innymi: type dynamic oraz obiekty tworzone przy użyciu interfejsu IDynamicMetaObjectProvider.

Poniższy przykład pokazuje, że zmiany wartości wewnątrz metody mają pływ na zmienne przekazne, jako parametry typu referencyjnego do takiej metody. 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {
            Test t1 = new Test();  // Utworzenie obiektu klasy Test
            t1.x = 10;
            t1.y = 20;

            // Przekazanie obiektu przez referencję
            Console.WriteLine("Przed wywołaniem metody: t1.x = {0}, t1.y = {1}\n", t1.x, t1.y);
            Zmien(ref t1);
            Console.WriteLine("Po wywołaniu metody: t1.x = {0}, t1.y = {1}", t1.x, t1.y);
        }

        // Metoda z parametrem typu referencyjnego
        static void Zmien(ref Test t)
        {
            t.x++;
            t.y++;
        }
    }

    // Klasa Test z polami x i y typu int
    class Test
    {
        public int x;
        public int y;
    }

//output:

Przed wywołaniem metody: t1.x = 10, t1.y = 20

Po wywołaniu metody: t1.x = 11, t1.y = 21

Parametry Typu Wyjściowego

Parametry typu wyjściowego (out parameters) to parametry, które służą do przekazywania wyników z metody do jej wywołującego. Różnią się one od innych parametrów tym, że nie muszą być przekazywane do metody jako argumenty przy jej wywołaniu. Zamiast tego, są one zainicjowane wartościami domyślnymi wewnątrz metody i służą do zwracania wyników.

Aby oznaczyć parametr jako typu wyjściowego, należy użyć słowa kluczowego out przed jego nazwą. 

                
using System;
{
    class Program
    {
        static void Main(string[] args)
        {

            int wynik;  // Zmienna, do której będzie przypisany wynik działania

            Podziel(15, 3, out wynik);   // Wywołanie metody z parametrem typu wyjściowego

            Console.WriteLine("Wynik: {0}", wynik);  // Wyświetlenie wyniku
        }

        // Metoda z parametrem typu wyjściowego
        static void Podziel(int a, int b, out int wynik)
        {
            wynik = a / b;
        }
    }
}

//output: 5

W metodzie nie trzeba przypisywać wartości domyślnej do parametru wynik, ponieważ będzie on zainicjowany wartością domyślną (0 dla typów numerycznych) wewnątrz metody.

Uwaga

parametry typu wyjściowego muszą być zainicjowane wewnątrz metody, a ich wartość musi być przypisana przed opuszczeniem metody. W przeciwnym razie program wygeneruje wyjątek typu InvalidOperationException.
